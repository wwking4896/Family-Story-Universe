# 完整 PRD：童話城堡 Fairy Castle

## 1. 產品定位

童話城堡是一座陪伴孩子長大的家庭童話王國。父母可以把孩子、家人、寵物與日常回憶轉化成持續成長的童話世界，並透過 AI 生成睡前故事、插圖與成長紀錄。

## 2. 願景與核心價值

童話城堡不是單次 AI 故事生成器，而是 Family Story Universe。產品長期價值來自：

- 專屬性：每個家庭都有自己的童話角色與世界觀。
- 連續性：故事會累積角色記憶、生活事件與成長歷程。
- 情感性：每篇故事都能對應真實家庭回憶。
- 長期保存：孩子長大後能回看童年故事。
- 商業延展：訂閱、年度童話書、AI 插圖、AI 語音與家族帳號。

## 3. 目標客群

### 主要使用者

- 25～45 歲父母。
- 有 2～8 歲孩子。
- 願意嘗試 AI 工具，重視親子陪伴、睡前故事與成長記錄。

### 次要使用者

- 阿公阿嬤、親戚、保母、幼兒園老師。
- 未來長大的孩子本人。

## 4. Persona

### Persona A：下班後想陪孩子的爸爸

- 情境：晚上想講故事但缺乏靈感。
- 需求：快速產生 3～5 分鐘、溫柔、適合睡前朗讀的故事。
- 成功條件：手機操作少於 5 個主要步驟即可產生故事。

### Persona B：想保留孩子成長事件的媽媽

- 情境：孩子第一次自己收玩具、第一次上學、第一次克服害怕。
- 需求：把事件轉成童話，不只是日記。
- 成功條件：故事保存在時光書，可依年月回顧。

### Persona C：想參與孫子成長的阿嬤

- 情境：不住在一起，但想知道孩子最近發生什麼事。
- 需求：未來可共同創作、留言、回顧家庭故事。
- MVP 處理：先建立 family_members 資料模型，邀請流程延後。

## 5. 核心使用情境

1. 睡前故事：選角色、場景、主題、長度，生成可朗讀故事。
2. 真實事件童話化：輸入今天事件，轉成溫柔的童話冒險。
3. 時光書回顧：依年月查看故事，作為年度童話書基礎。
4. 長大後共同創作：孩子未來加入自己的角色與想法。

## 6. 使用者旅程

1. Landing Page：理解產品價值。
2. 註冊 / 登入：建立帳號。
3. 建立家庭：命名童話城堡。
4. 建立孩子：輸入暱稱、生日、可選性別與頭像。
5. 建立角色：設定童話名字、角色類型、喜好、害怕、魔法能力。
6. 生成故事：選主角、區域、主題、長度、語氣，可填真實事件。
7. 故事閱讀：朗讀、收藏、回到列表。
8. 時光書：回顧過去故事。

## 7. 商業模式

| 方案 | 定位 | MVP 處理 |
|---|---|---|
| Free | 快速體驗 | 預留 plan_type、quota 欄位 |
| Premium | 個人家庭訂閱 | 預留訂閱與用量資料表 |
| Family | 多家人共同創作 | 預留 family_members 權限 |
| 年度童話書 | 高情感價值商品 | MVP 先完成時光書資料基礎 |
| AI 語音 | V2/V3 加值 | MVP 不做 |

## 8. 成功指標

- 註冊到第一篇故事完成率 >= 50%。
- 第一篇故事產生平均時間 <= 60 秒。
- 7 日內回訪並再次生成故事比例 >= 25%。
- AI JSON schema 合法率 >= 95%。
- 內容安全通過率 >= 98%。
- 家庭資料越權讀取事件：0。

## 9. 非功能需求

- 安全：JWT、密碼雜湊、家庭資料隔離、audit log。
- 隱私：兒童資料最小化、性別選填、軟刪除與資料匯出/刪除規劃。
- 可用性：手機優先，表單步驟短，錯誤訊息友善。
- 可維護性：Clean Architecture、AI Provider 抽象、OpenAPI 文件。
- 可觀測性：AI job 狀態、usage log、error log、基本健康檢查。

## 10. 風險與緩解

| 風險 | 影響 | 緩解 |
|---|---|---|
| AI 內容不安全 | 高 | schema 驗證、內容安全規則、二次 safety check |
| AI 成本失控 | 中高 | quota、rate limit、usage log |
| 兒童資料隱私 | 高 | 權限隔離、最小化欄位、audit log |
| 故事品質不穩 | 中 | prompt template、AI eval、可重試機制 |
| MVP 過大 | 中 | 嚴格延後語音、實體書、社群、遊戲化 |
